### 1.1 Egg 特点

- 整合了后端技术，提供一站式开发体验
- 指定了一套规范，同意了开发模式
- 提供了各种插件，具备灵活的扩展能力
- 约定大于配置
- 内置多进程管理
- 使用 MVC 模式

### 1.2 使用Egg搭建工程

#### 方式 1：使用脚手架

```javascript
npx create-egg@beta --template tegg hackernews-tegg

cd hackernews-tegg
npm install
```

#### 方式 2：手动搭建

```javascript
npm i egg // 安装 egg 核心库
npm i @eggjs/bin --save-dev // 安装 egg 命令行工具
```

添加 `npm scripts` 到 `package.json`：

```json
{
  "name": "hackernews-tegg",
  "scripts": {
    "dev": "egg-bin dev"
  }
}
```

目录结构如下

```javascript
egg-example
├── app
│   ├── controller
│   │   └── home.js
│   └── router.js
├── config
│   └── config.default.js
└── package.json
```

编写 Controller

如果你熟悉 Web 开发或 MVC，肯定猜到我们第一步需要编写的是 [HTTP Controller](https://www.eggjs.org/zh-CN/basics/httpcontroller)。

```plain
// app/controller/home.js
const Controller = require('egg').Controller;

class HomeController extends Controller {
  async index() {
    this.ctx.body = 'Hello world';
  }
}

module.exports = HomeController;
```

配置路由映射：

```plain
// app/router.js
module.exports = (app) => {
  const { router, controller } = app;
  router.get('/', controller.home.index);
};
```

加一个[配置文件](https://www.eggjs.org/zh-CN/basics/config)：

```plain
// config/config.default.js
exports.keys = '<此处改为你自己的 Cookie 安全字符串>';
```

好，现在可以启动应用来体验下：

```plain
npm run dev
open http://localhost:7001
```

运行流程

![img](https://cdn.nlark.com/yuque/0/2026/jpeg/21504715/1774622658688-06c56328-ee2d-4c76-9a2d-2c3d5d8b8a01.jpeg)

- 生成配置：egg会在启动时读取`config`文件夹中的配置，以及`app/router.js`中的路由信息，然后将最终的配置生成到`run`文件夹中
- 启动egg：egg会在内部创建`Koa`实例，并作出适当的初始化工作，然后监听`7001`端口（默认）
- 匹配路由：egg在内部使用了`@koa/router`，会根据路由表中请求的路径和方法，把请求交给指定的`action`进行处理
- 请求处理：egg遵循`MVC`模式，**请求始终是交给**`**Controller**`**中的**`**Action**`**进行处理**

- `Controller`表现为一个类，继承自`egg`中的`Controller`
- `Action`表现为一个`Controller`中的实例方法，用于处理请求
