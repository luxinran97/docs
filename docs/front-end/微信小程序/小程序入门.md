##  一、文件目录结构介绍

一个完整的小程序项目分为两部分：主体文件、页面文件

<img src="/工程结构.png" width="460px" height="500px">

主体文件又称全局文件，作用于整个小程序，必须放到项目的根目录下，有三部分组成：

- app.js 小程序入口文件
- app.json 小程序的全局配置文件
- app.wxss 小程序的全局样式

> 主体文件名称必须是 app，且前两个是必须的

------

页面文件是每个页面所需的文件，一个页面一个文件夹

每个页面通常由四个文件组成，每个文件只对当前页面有效：

- .js 页面逻辑
- .wxml 页面结构
- .wxss 页面样式
- .json 小页面配置

### 3.1 尺寸单位

- rpx（responsive pixel）: 可以根据屏幕宽度进行自适应。规定任何屏幕宽为750rpx。如在 iPhone6 上，屏幕宽度为375px，共有750个物理像素，则750rpx = 375px = 750物理像素，1rpx = 0.5px = 1物理像素。

- 其他单位不能实现自适应

### 3.4 navigator跳转组件

在小程序中跳转需要使用 navigator 组件，常用属性有两个

1. url：当前小程序内的跳转链接
2. Open-type：跳转方式
   1. navigate：保留当前页面，跳转到应用内某个界面。但是不能跳到 tabbar 界面
   2. redirect：关闭当前页面，跳转到应用内某个页面，但是不能跳到 tabbar 界面
   3. switchTab：只能跳转到 tabbar 页面，并关闭其他非 tabBar 页面
   4. reLanch：关闭所有页面，打开到应用内的某个页面
   5. navigateBack：关闭当前页面，返回上一页面或多级页面(delta 参数控制)

> 注意事项：
>
> 1. 路径后可以带参数。参数与路径之间使用？分割，参数键值使用=相连，不同参数用&分隔
> 2. openType="switchTab"时不支持传参

```xml
      <text>鲜花玫瑰</text>

```

## 四、事件系统

### 4.1 事件绑定和事件对象

小程序绑定事件与网页开发绑定事件几乎一致，只不过在小程序不能通过 on 的方式绑定事件，也没有 click 等事件，小程序中绑定事件使用 bind 方法，click 事件也需要使用tap 事件来代替，绑定事件的方式有两种：

1. bind 事件名，bind 后面需要跟上冒号，冒号后面跟上事件名，例如`<view bind:tap="fnName"></view>`

   ```js
   // js 文件
   Page({
     handle(){
       console.log("事件触发");
     }
   })
   ```

   

2. 去掉冒号即可

### 4.2 事件分类以及阻止事件冒泡

事件分为：冒泡事件和非冒泡事件

冒泡：事件触发后，该事件会向父节点传递

> 使用 bind 绑定的事件，会触发事件冒泡，如果想阻止，可以使用catch 来绑定事件

`<button catch:tap="btnHandler"></button>`

### 4.3 事件传参 data-自定义数据

事件传参：在触发事件时，将一些数据作为参数传递给事件处理函数

组件上通过 `data-*`的方式定义需要传递的数据，其中*是自定义的属性，例如`<view data-id="100" bindtap="handler" />`

然后通过事件对象 进行获取自定义数据

// currentTarget 事件绑定者

// target 事件触发者

```js
 handle(e){
    console.log("事件触发", e.currentTarget.dataset.id);
 },
```

> 注意：`data-id-name="100"`这种中划线被转化成了小驼峰
>
> 如果是小驼峰，在事件对象中被转化为全小写 

### 4.4 事件传参-mark-自定义数据

mark 是一种自定义属性，可以在组件上添加，用于识别具体触发事件的 target 热点。同时 mark 还可以用于承载一些自定义数据

```xml
<button bind:tap="handler" mark:id="1" mark:name="tom">
  按钮
</button>
```

```js
handler(event){
  // 会获取到触发事件的节点以及父节点身上所有的 mark 数据
    console.log(event.mark.id, event.mark.name);
 },
```

## 五、wxml语法

### 5.1 声明和绑定数据

小程序页面使用的数据均需要在 Page 方法的 data 对象中进行声明定义

将数据声明好以后，在 WXML 中使用双大括号{{}}将变量包起来，从而将数据绑定

在{{}}内部可以进行一些简单运算：算术运算、三元运算、逻辑判断、其他...

> 注：在{{}}内部，只能写表达式，不能写语句，也不能调 js 方法

### 5.2 setData()修改数据

小程序中修改数据不推荐通过赋值方式进行修改，通过赋值的方式修改数据无法改变页面数据

而是要通过 setData()方法进行修改，setData()方法接受对象作为参数，key 是要修改的数据，value 是值

setData 作用：1. 更新数据 2. 驱动视图更新

```js
  data: {
    name: "张三",
  },
    
  handler(event){
    console.log(event.mark.id, event.mark.name);
    this.setData({name: "李四"});
    console.log(this.data.name);
  },
```

### 5.3 setData修改对象类型数据

```js
updateObj(){
    // 新增，修改属性
    // this.setData({
    //   // 如果给对象新增属性，可以将 key 写成数据路径的方式 a.b.c
    //   'userInfo.age':18,
    // }),

    // 新增修改多个, 借用 ES6 语法
    const userInfo = {
      ...this.data.userInfo,
      name:"larry",
      age:19,
    }
    this.setData({
      // 如果给对象新增属性，可以将 key 写成数据路径的方式 a.b.c
      userInfo,
    })
  },
```

// 删除数据

```js
// 删除单个属性
delete this.data.userInfo.age;
this.setData({userInfo: this.data.userInfo})

// 删除多个, es6 的 rest 剩余参数
const {age, test, ...rest} = this.data.userInfo;
this.setData({userInfo: rest})
```

### 5.4 修改数组类型

```js
Page({
  data: {
    list: [
      {id: 1, name: '项目1', completed: false},
      {id: 2, name: '项目2', completed: true}
    ]
  },

  // 添加新项目
  addNewItem() {
    const newItem = {
      id: Date.now(),
      name: '新项目',
      completed: false
    }
    
    this.setData({
      list: [...this.data.list, newItem]
    })
  },

  // 切换完成状态
  toggleComplete(index) {
    this.setData({
      [`list[${index}].completed`]: !this.data.list[index].completed
    })
  },

  // 删除项目
  deleteItem(index) {
    this.setData({
      list: this.data.list.filter((item, i) => i !== index)
    })
  },

  // 更新项目名称
  updateItemName(index, newName) {
    this.setData({
      [`list[${index}].name`]: newName
    })
  },

  // 清空数组
  clearList() {
    this.setData({
      list: []
    })
  }
})
```

### 5.5 简易数据双向绑定

如果希望用户输入数据的同时改变 data 中的数据，可以借助简易双向绑定机制。在对应的属性前加 model:前缀即可

例如`<input model:value="{{value}}"/>`

> 注：简易数据双向绑定有如下限制
>
> 1. 只能是一个单一字段的绑定，不能拼接,例如`<input model:value="值为{{value}}"/>`
> 2. 尚不能写 data 路径，也就是不支持数组和对象

```js
<input type="text" model:value="{{value}}"/>
<view>{{value}}</view>
```

### 5.6 渲染列表

在微信小程序中，WXML 提供了 `wx:for` 指令来渲染列表数据。

每一项的变量名默认为 item，下标变量名默认为 index；

在使用 wx:for 进行遍历时，建议加上 wx:key 属性，两种形式

1. 字符串：代表需要遍历的 array 中 item 的的某个属性，该属性是列表中唯一的字符串或数字，且不能动态改变
2. 保留关键字*this 代表在 for 循环中的 item 本身，当 item本身是一个唯一的字符串或数字时可以使用

> 注意事项：
>
> 1. 如果不加 wx:key,会报 warning，如果明确知道该列表是静态，即数据不会改变，可以忽略
> 2. 在给 wx:key 添加属性值的时候，不需要使用双大括号，直接使用遍历的 array 中 item 的某个属性

```js
<view wx:for="{{numList}}" wx:key="index">{{item}}-{{index}}</view>

<view wx:for="{{fruitList}}" wx:key="id">{{item.name}}-{{index}}</view>
```

------

进阶用法

1. 自定义索引和变量名

```js
<view wx:for="{{users}}" wx:for-index="idx" wx:for-item="user" wx:key="id">
  <text>索引: {{idx}}</text>
  <text>姓名: {{user.name}}</text>
  <text>年龄: {{user.age}}</text>
</view>
```

2. 在微信小程序中，`<block>` 是一个包装元素，不会在页面中实际渲染，只用于接收控制属性。

- 使用 block 包装元素

  ```js
  <block wx:for="{{items}}" wx:key="id">
    <view class="item">
      <text>{{item.name}}</text>
    </view>
    <view class="divider"></view>
  </block>
  ```

- 不使用

  ```js
  <!-- 不使用 block，每个元素都需要 wx:for -->
  <view wx:for="{{items}}" wx:key="id" class="item">
    <text>{{item.name}}</text>
  </view>
  <view wx:for="{{items}}" wx:key="id" class="divider"></view>
  ```

  1. **减少重复代码**：避免在每个元素上都写 `wx:for`
  2. **结构清晰**：逻辑相关的元素可以组织在一起
  3. **无额外渲染**：`<block>` 不会生成实际 DOM 节点
  4. **灵活性**：可以自由组合多个元素

## 六、小程序API

小程序开发框架提供丰富的微信原生 API，可以方便的调起微信提供的能力，如获取用户信息，本地存储，支付功能等。详细介绍请参考 [API 文档](https://developers.weixin.qq.com/miniprogram/dev/api/index.html)。

通常，在小程序 API 有以下几种类型：

1. **事件监听 API** 

我们约定，以 `on` 开头的 API 用来监听某个事件是否触发，如：[wx.onSocketOpen](https://developers.weixin.qq.com/miniprogram/dev/api/network/websocket/wx.onSocketOpen.html)，[wx.onCompassChange](https://developers.weixin.qq.com/miniprogram/dev/api/device/compass/wx.onCompassChange.html) 等。

这类 API 接受一个回调函数作为参数，当事件触发时会调用这个回调函数，并将相关数据以参数形式传入。

2. **同步 API**

我们约定，以 `Sync` 结尾的 API 都是同步 API， 如 [wx.setStorageSync](https://developers.weixin.qq.com/miniprogram/dev/api/storage/wx.setStorageSync.html)，[wx.getSystemInfoSync](https://developers.weixin.qq.com/miniprogram/dev/api/base/system/wx.getSystemInfoSync.html) 等。此外，也有一些其他的同步 API，如 [wx.createWorker](https://developers.weixin.qq.com/miniprogram/dev/api/worker/wx.createWorker.html)，[wx.getBackgroundAudioManager](https://developers.weixin.qq.com/miniprogram/dev/api/media/background-audio/wx.getBackgroundAudioManager.html) 等，详情参见 API 文档中的说明。

同步 API 的执行结果可以通过函数返回值直接获取，如果执行出错会抛出异常。

3. 异步 API

大多数 API 都是异步 API，如 [wx.request](https://developers.weixin.qq.com/miniprogram/dev/api/network/request/wx.request.html)，[wx.login](https://developers.weixin.qq.com/miniprogram/dev/api/open-api/login/wx.login.html) 等。这类 API 接口通常都接受一个 `Object` 类型的参数，这个参数都支持按需指定以下字段来接收接口调用结果：

**Object 参数说明**

| 参数名   | 类型     | 必填 | 说明                                             |
| :------- | :------- | :--- | :----------------------------------------------- |
| success  | function | 否   | 接口调用成功的回调函数                           |
| fail     | function | 否   | 接口调用失败的回调函数                           |
| complete | function | 否   | 接口调用结束的回调函数（调用成功、失败都会执行） |
| 其他     | Any      | -    | 接口定义的其他参数                               |

### 6.1 发起网络请求

```js
getData(){
    wx.request({
      url: 'url',
      method:'GET',
      data:{},
      header:{},
      success:(res)=>{
        console.log(res);
        this.setData();
      },
      fail:(err)=>{
        console.log(err);
      },
      // 相当于 finally
      complete:(res)=>{
        console.log(res);
      }
    })
  },
```

### 6.3 本地存储

每个微信小程序都可以有自己的本地缓存，可以通过 [wx.setStorage](https://developers.weixin.qq.com/miniprogram/dev/api/storage/wx.setStorage.html)/[wx.setStorageSync](https://developers.weixin.qq.com/miniprogram/dev/api/storage/wx.setStorageSync.html)、[wx.getStorage](https://developers.weixin.qq.com/miniprogram/dev/api/storage/wx.getStorage.html)/[wx.getStorageSync](https://developers.weixin.qq.com/miniprogram/dev/api/storage/wx.getStorageSync.html)、[wx.clearStorage](https://developers.weixin.qq.com/miniprogram/dev/api/storage/wx.clearStorage.html)/[wx.clearStorageSync](https://developers.weixin.qq.com/miniprogram/dev/api/storage/wx.clearStorageSync.html)，[wx.removeStorage](https://developers.weixin.qq.com/miniprogram/dev/api/storage/wx.removeStorage.html)/[wx.removeStorageSync](https://developers.weixin.qq.com/miniprogram/dev/api/storage/wx.removeStorageSync.html) 对本地缓存进行读写和清理。

隔离策略

同一个微信用户，同一个小程序 storage 上限为 10MB。storage 以用户维度隔离，同一台设备上，A 用户无法读取到 B 用户的数据；不同小程序之间也无法互相读写数据。

**插件隔离策略**

1. 同一小程序使用不同插件：不同插件之间，插件与小程序之间 storage 不互通。
2. 不同小程序使用同一插件：同一插件 storage 不互通。

清理策略

本地缓存的清理时机跟代码包一样，只有在代码包被清理的时候本地缓存才会被清理。

url参数传递

EventChannel 事件通信

```js
// 页面A - 发送数据
wx.navigateTo({
  url: '/pages/pageB/pageB',
  events: {
    // 监听页面B发送的事件
    onDataUpdate: (data) => {
      console.log('收到页面B的数据:', data)
      this.setData({
        updatedData: data
      })
    }
  },
  success: (res) => {
    // 向页面B发送数据
    res.eventChannel.emit('onLoadData', {
      userId: '123',
      userName: '张三'
    })
  }
})

// 页面B - 接收和发送数据
Page({
  onLoad(options) {
    // 获取事件通道
    const eventChannel = this.getOpenerEventChannel()
    
    // 监听页面A发送的事件
    eventChannel.on('onLoadData', (data) => {
      console.log('收到页面A的数据:', data)
      this.setData({
        userInfo: data
      })
    })
    
    // 向页面A发送数据
    this.sendDataToPageA = () => {
      eventChannel.emit('onDataUpdate', {
        status: 'success',
        message: '操作完成'
      })
    }
  }
})
```

### 6.5 页面处理函数-上拉加载

方式：

1. 在 app.json 或者 page.json 中配置距离页面底部距离：onReachBottomDistance；默认 50px
2. 在页面 js 中定义 onReachBottom 事件监听用户上拉加载

```js
onReachBottom() {
    console.log("监听用户上拉加载");
    const lastNum = this.data.numList[this.data.numList.length - 1];
    const newArr = [lastNum + 1, lastNum + 2, lastNum + 3];
    this.setData({
      numList: [...this.data.numList, ...newArr]
    })
  },
```

### 6.6 页面处理函数-下拉刷新

方式：

1. 在 app.json 或者 page.json 中开启允许下拉，同时可以配置窗口，loading 样式等
2. 在页面 js 中定义 onPullDownRefresh 事件监听用户下拉刷新

```js
onPullDownRefresh() {
   
    // 刷新数据的代码块
    // ...
    // 下拉刷新数据后自动回弹  将loading的样式取消
    wx.stopPullDownRefresh()
},
```

### 7.2 组件的数据以及方法

组件的数据和方法需要在组件.js 的 Component 方法中进行定义

1. data：定义组件的内部数据
2. methods：在组件中事件处理程序需要写到 methods 中才行

```javascript
Page({
   data: {
    isChecked:false
  },

  methods:{
    updateChecked(){
      this.setData({isChecked: !this.data.isChecked})
    },
  },
})
```

### 7.3 组件属性

Properties 是指组件的对外属性，主要用来接收组件使用者传递给组件内部的数据，和 data 一同用于组件的模板渲染

```xml
// index.wxml
<custom-checkbox label="我已阅读并同意 用户协议 和 隐私协议" position="right"> </custom-checkbox>

<view class="line"></view>

<custom-checkbox label="匿名提交" position="left"> </custom-checkbox>
```

```js
// custom-checkbox.js
properties: {
    label: {
      type: String,
      value: ''
    },

    position: {
      type: String,
      value: 'right'
    }
  }
```

```html
// custom-checkbox.wxml
<view class="custom-checkbox-container">
  <view class="custom-checkbox-box {{position === 'right'? 'right':'left'}}">
    <checkbox checked="{{isChecked}}" bind:tap="updateChecked" />

    <view><text>{{label}}</text></view>
  </view>
</view>
```

> 注：js 中可以访问和获取 properties 中的数据，但是一般情况下不建议修改，会造成数据流混乱
